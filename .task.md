## Implementation plan (for whoever picks this up in get-me-job)

Once re-rooted in `/Users/bentomac/get-me-job/`:

### Approach

`scanner-form.tsx` in get-me-job already has a JD URL scrape input + a `Paste job description (optional)` textarea wired into `scanResume(profile, resumeText, job)`. `analyzeKeywords()` in `lib/ats/analyzer.ts` already produces `KeywordAnalysis[]` with `found: boolean`, synonym matching, and a match rate.

So "ATS Scanner currently scores resume in isolation" is half-wrong: the JD path exists, but its results are buried inside `ScoreDisplay` / scoring axes; the user gets no foreground match% card or missing-keyword chips. Build the **presentation layer** + a self-contained, testable JD keyword extractor.

### Files to create

- **`apps/web/src/lib/ats/jd-keywords.ts`** — `extractJDKeywords(jdText, opts?: { max?: number }): JDKeyword[]` where `JDKeyword = { term, weight, source: "requirement" | "body" }`. Tokenize → drop stopwords → score by frequency × section weight (Requirements 2x, body 1x). Bigram/trigram detection. Cap at `max ?? 30`. Pure function. Mirror patterns in `text-to-profile.ts:292-388`.
- **`apps/web/src/lib/ats/match-score.ts`** — `computeJDMatch(resumeText, jdKeywords): { matchPct, matched, missing }`. Word-boundary regex on `normalizeText(resumeText)`, synonym fallback via `getSynonyms()` at `SYNONYM_MATCH_WEIGHT = 0.8` from `synonyms.ts`. `matchPct = round(100 * sum(matched weight × matchTypeWeight) / sum(all weight))`. Sort `missing` by weight desc.
- **`apps/web/src/components/ats/jd-match-card.tsx`** — `"use client"`. Props: `{ result: JDMatchResult }`. Header: "Match against this job" + match% (large, color via existing threshold logic) + thin `<Progress />` bar. Below: "Missing from your resume" + chips for `result.missing.slice(0, 10)`. Chip click → `navigator.clipboard.writeText(term)` + `addToast({ type: "success", title: "Copied: " + term })`. Empty state when `missing.length === 0`. `aria-label="Copy keyword <term>"` on chip buttons. Defensive `if (!navigator.clipboard)` branch.

### Files to modify

- **`apps/web/src/components/ats/scanner-form.tsx`** — JD textarea already exists. Import `<JDMatchCard>`, `computeJDMatch`, `extractJDKeywords`. In the `if (result)` block, after the scoring-axes grid and before `<FixSuggestionsList>`, conditionally render `<JDMatchCard>` when `jobText.trim().length >= 20`. `useMemo` keyed on `[resumeText, jobText, parsedProfile]` for the match computation. Pass `parsedProfile?.rawText ?? resumeText` for matching.
- **`apps/web/src/app/[locale]/(marketing)/ats-scanner/page.tsx`** — likely no edit needed; the page is a thin shell. Verify with type-check after the form changes land.
