# IMPROVE-03: Opportunities — auto-status from Gmail inbox

Implement spec at `docs/audit/feature-specs/2026-05-10-improve-03-auto-status-from-gmail.md`.

Use the already-wired Gmail integration to scan inbox for status-changing emails and auto-update opportunities (saved → applied → interviewing → rejected/offer).

Cron-triggered daily 9 AM UTC. Endpoint: `/api/cron/gmail/status-detect`.

Status patterns (regex-based, 80% accuracy target):
- "Thank you for applying" → applied
- "schedule a call" / "interview" / "next round" → interviewing
- "Unfortunately" / "decided to move forward" → rejected
- "extend an offer" / "offer letter" → offer

New/modified files:
- `apps/web/src/lib/email/status-patterns.ts` (new) — regex patterns
- `apps/web/src/lib/email/match-opportunity.ts` (new) — fuzzy company match
- `apps/web/src/app/api/cron/gmail/status-detect/route.ts` (new) — cron handler
- `apps/web/src/lib/db/schema.ts` — add `gmail_auto_status_enabled`, `gmail_last_scanned_at` to user_settings
- `apps/web/src/app/[locale]/(app)/settings/page.tsx` — toggle UI
- `apps/web/vercel.json` — add `{ "path": "/api/cron/gmail/status-detect", "schedule": "0 9 * * *" }`

Acceptance:
- Settings toggle, default OFF (privacy)
- Status only ADVANCED, never reverted
- Match by company name (fuzzy)
- Notification on auto-update with "Undo" button
- Test fixtures: 5 sample emails (1 per status + 1 false-positive)
- type-check + lint clean

Out of scope: LLM email classification, multi-language, attachment reading.
