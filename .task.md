# IMPROVE-04: Interview Prep — live feedback during mock

Implement spec at `docs/audit/feature-specs/2026-05-10-improve-04-interview-live-feedback.md`.

Interview Prep transcribes user answers but gives no feedback. Add a heuristic coaching layer with per-answer scorecards + end-of-session aggregate.

Metrics:
1. Filler word count (um, like, you know, etc)
2. STAR coverage (keyword-based detection of Situation/Task/Action/Result)
3. Quantification count (numbers/metrics in answer)
4. Length vs target (60-90s behavioral, 90-180s technical)
5. Pace (words per minute, target 130-150)

New files:
- `apps/web/src/lib/interview/feedback.ts` — analyze transcript → scorecard
- `apps/web/src/lib/interview/star-detector.ts` — keyword-based STAR detection
- `apps/web/src/lib/interview/filler-words.ts` — count fillers
- `apps/web/src/components/interview/answer-feedback-card.tsx` — per-answer scorecard
- `apps/web/src/components/interview/session-feedback-summary.tsx` — end-of-session aggregate

Modify:
- `apps/web/src/components/interview/interview-active-session.tsx` — show feedback after each answer

Acceptance:
- Scorecard appears below each answered question
- All 5 metrics visible with color-coded thresholds (green/yellow/red)
- Top suggestion text (single most actionable feedback)
- End-of-session aggregate report
- Test fixtures: 3 sample answers (strong/medium/weak) → assert correct ratings
- type-check + lint clean

Out of scope: LLM-based feedback, body language webcam, real-time during-answer indicators.
