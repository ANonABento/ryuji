# Custom command builder — owner-defined aliases

`/customcmd add <name> <response>`
- Defines a new alias command owned by the bot owner.
`/customcmd list`
- Lists all custom commands.
`/customcmd delete <name>`
- Removes a command by name.

Aliases are triggered by typing `/<name>` through dynamic command registration. Responses are persisted in SQLite. Behavior is owner-only.

Acceptance: `add`, `list`, and `delete` work end-to-end; custom aliases respond correctly; `npx tsc --noEmit` passes.
